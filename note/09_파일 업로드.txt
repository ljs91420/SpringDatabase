# 클라이언트에서 서버로 파일 업로드하기

	- 클라이언트로부터 업로드된 파일은 서버에 저장해둘 수 있다.
	- DB에 파일을 그대로 보관하는 것은 많은 비용이 발생하므로 DB에는 업로드한 파일의 경로만 보관하는 것이 보편적이다.
	- 파일 업로드시 클라이언트가 정한 이름은 겹칠 위험이 있으므로 실제로 파일을 보관할 때는 이름이 겹치치 않도록 조치를 취해야 한다.
	  (시간을 활용하거나 또는 랜덤으로 생성된 ID를 부여)
	- 업로드했던 파일을 사용자가 다시 다운 받기를 원할 때 업로드할 때 사용한 이름을 그대로 제공하기 위해 원본의 이름도 보관해야 한다.
	  
# 파일 업로드 폼 만들기

	- method는 POST로 설정하고 추가로 enctype 속성을 multipart/form-data로 설정해야 한다.
	- 파일이 포함된 폼 데이터를 바인딩받는 DTO는 해당 파일을 MultipartFile 타입으로 받을 수 있다.
	- 파일이 여러 개 포함된 폼 데이터를 바인딩 받는 DTO는 해당 파일들을 List<MultipartFile> 타입으로 받을 수 있다.
